# 岗位方向标注口径

更新时间：2026-05-13

## 标准类目

- `前端开发`
- `后端开发`
- `测试开发`
- `算法工程`
- `AI应用开发`

## 冻结边界规则

### 1. `应用算法工程师 / 算法应用 / 大模型应用算法`

默认标为 `AI应用开发`。

适用标题示例：

- `大模型应用算法工程师`
- `算法应用工程师`
- `大模型算法应用`
- `copilot-大模型应用算法工程师`

只有在标题或职责明显强调以下内容时，才改标 `算法工程`：

- `预训练`
- `后训练`
- `蒸馏`
- `推理优化 / 推理加速`
- `模型训练`
- `强化学习`
- `算法研究员`

### 2. `AI应用后台开发 / Agent后台开发 / RAG后台开发`

默认标为 `后端开发`。

原因：岗位职责核心是服务端系统、框架、平台、调度、稳定性、性能与工程交付，不因为接入 LLM / Agent / RAG 就改成 `AI应用开发`。

### 3. `Agent应用开发 / AI Agent研发工程师 / LLM Application`

默认标为 `AI应用开发`。

原因：岗位目标是把模型能力接进业务、产品、工作流或交互体验，而不是做底层训练或推理优化。

### 4. `训练 / 推理 / 后训练 / 强化学习 / 算法研究`

默认标为 `算法工程`。

适用标题示例：

- `大模型训练工程师`
- `推理性能优化工程师`
- `后训练算法工程师`
- `强化学习算法研究员`
- `多模态大模型算法工程师`

## 非边界直判

- 标题含 `前端 / web前端 / 客户端前端`，优先标 `前端开发`
- 标题含 `测试 / QA / 自动化测试 / 性能测试`，优先标 `测试开发`
- 标题含 `后台 / 后端 / 服务端 / 存储 / 框架研发 / 平台研发`，优先标 `后端开发`
